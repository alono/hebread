## 1. רקע ובעיה

ילד שסיים כיתה ב' וכמעט לא קורא. אין קושי קוגניטיבי — יש פער שנוצר מנסיבות (כיתה א' קשה, מעבר בית ספר). שיעורים פרטיים עובדים אבל יקרים. נדרש כלי ביתי, חינמי, שמאפשר לילד להתקדם מהר בקצב שלו — כולל **דילוג מהיר על שלבים שהוא כבר שולט בהם** — מהצליל הבודד ועד קריאת פסקאות שלמות ברצף.

### עקרונות פדגוגיים (מבוססי מחקר — אין לסטות מהם)

1. **שיטה פונטית-אנליטית שיטתית ומדורגת** — זו הגישה הרשמית של משרד החינוך בישראל, והיא מגובה מחקרית. מלמדים קשר אות+ניקוד→צליל באופן מפורש, מובנה ומדורג. לא "מילה שלמה", לא ניחוש מהקשר.
2. **עברית מנוקדת בלבד בכל שלבי המערכת** — הניקוד הופך את העברית לכתב "שקוף" (כל סימן = צליל אחד), ושקיפות מקלה משמעותית על רכישת קריאה.
3. **מודעות פונמית לפני הכול** — הקושי המרכזי של ילדים מתקשים הוא הבחנה ותרגול של צלילים. לכן תרגיל הליבה הוא: *שומעים צליל → בוחרים את הכתיב הנכון* (המכניקה המוכחת של GraphoGame, שנבדקה ב-RCTs במדינות רבות).
4. **מעט וקצר, הרבה פעמים** — סשנים של 10–15 דקות. עדיף כל יום 10 דקות מפעם בשבוע שעה.
5. **חזרתיות מרווחת (spaced repetition קל)** — פריט שנענה בטעות חוזר לתור שוב ושוב עד שליטה. פריטים שנשלטים מופיעים פחות.
6. **אוטומטיזציה של מילים שכיחות** — חזרה אינטנסיבית על מילים נפוצות בונה קריאה אוטומטית ומפנה קשב להבנה.
7. **חיבור לטקסט אמיתי** — מחקרי GraphoGame מראים שרווחים "בתוך המשחק" לא תמיד עוברים לקריאת טקסט. לכן חובה שהשלבים הגבוהים כוללים קריאת משפטים ופסקאות אמיתיים בקול רם, עם אישור הורה.
8. **אפס תסכול, אפס השפלה** — טעות אף פעם לא "מפסידה". טעות = הצליל מושמע שוב + הפריט חוזר אחר כך. אין טיימר לוחץ בשלבי הקנייה (טיימר קיים רק בסבבי "טורבו" אופציונליים).

## 2. חזון המוצר

אפליקציית ווב סטטית (ללא backend), קוד פתוח (MIT), רצה על Vercel, עברית RTL מלאה, מותאמת טאבלט/מובייל/דסקטופ. ילד בן 7–9 פותח לבד, שומע צלילים, משחק, צובר כוכבים, ומטפס בסולם שלבים — מ-**אָ אַ** ועד **קריאת פסקה שלמה ברצף**. הורה רואה התקדמות ומאשר קריאה בקול.

**לא במוצר (Non-goals ל-v1):** זיהוי דיבור (speech-to-text), חשבונות משתמשים, שרת, מסד נתונים, תשלומים, ריבוי שפות, כתיבה במחברת. הכתיבה נתמכת רק כ"הרכבת מילים" מהברות.

### מנגנון "רצים מהר" — קריטי לילד עם פער
- **מבחן קפיצה (Skip Test):** בכניסה לכל שלב מוצג כפתור "⚡ אני כבר יודע!". 10 פריטים אקראיים מהשלב; 9/10 נכונים ← השלב נחשב הושלם (מסומן ⚡), עוברים הלאה מיד.
- **קידום אדפטיבי בתוך שלב:** רצף של 8 תשובות נכונות ראשונות ← קיצור השלב (מדלגים על מחצית הפריטים שטרם הוצגו).
- **אין נעילה קשיחה:** אפשר תמיד "להציץ" בשלב הבא; ההתקדמות הרשמית נשמרת לפי השלמות.

## 5. סוגי תרגילים (Game Mechanics)

כל שלב מורכב מ"סיבובים" של 8–12 פריטים. כל פריט הוא אחד מהתרגילים הבאים, לפי מה שרלוונטי לשלב:

1. **🎧 שמע-ובחר (הליבה, שלבים 1–8):** מושמע צליל ("בָּה!") ומוצגות 3–4 קלפים ענקיים (בָּ / בִּ / דָ / כָּ). המסיחים דומים פונטית או ויזואלית (ב/כ, ד/ר, ח/ה, קמץ/פתח מול חיריק). לחיצה נכונה: אנימציית קונפטי קטנה + הדמות מטפסת שלב בסולם. שגויה: הצליל מושמע שוב, הקלף הנכון מהבהב, הפריט חוזר לתור.
2. **👆 ראה-והשמע (שלבים 1–6):** מוצגת הברה ענקית, הילד לוחץ עליה ושומע אותה — מסך "היכרות" בתחילת כל קבוצת צלילים חדשה (מקביל ל-אאאא בָּבָּבָּ המסורתי: לחיצה חוזרת משמיעה שוב ושוב).
3. **🧩 הרכב מילה (שלבים 7–8):** נשמעת מילה + תמונה; גוררים/מקישים הברות לפי הסדר (אַ + בָּ + א ← אַבָּא).
4. **🖼️ מילה-לתמונה (שלבים 7–9):** מוצגת מילה/משפט מנוקדים, בוחרים תמונה אחת מ-3 (תמונות = אימוג'י גדולים ב-v1, בלי נכסים גרפיים).
5. **⚡ סבב טורבו (אופציונלי, כל שלב שהושלם):** 60 שניות, כמה שיותר פריטים — בונה מהירות ואוטומטיות. שיא אישי נשמר.
6. **📖 קריאה בקול עם הורה (שלבים 9–12):** הטקסט מוצג בפונט ענק, ההורה מסמן ✓/✗ לכל משפט. בסוף: שאלת הבנה (בחירה מ-3, מוקראת בשמע). זהו הגשר לטקסט אמיתי — חובה, לא תוספת.

### שכבת המוטיבציה
- **דמות-מלווה:** ינשוף/דרקון קטן (SVG, ללא IP קיים) שמטפס בסולם עם כל תשובה נכונה — ובסוף סיבוב מגיע ל"ענן פרס".
- **כוכבים:** 1–3 כוכבים לסיבוב לפי דיוק (לא לפי מהירות!). כוכבים מצטברים פותחים "מדבקות" בלוח אוסף (SVG פשוטים).
- **רצף יומי (Streak):** שמש קטנה לכל יום עם סשן ≥5 דקות.
- **מפת שלבים:** מסך הבית הוא שביל/סולם עם 12 תחנות, מראה בדיוק איפה הילד ומה נפתח.
- **בלי:** לוחות מובילים, חשבונות, כישלון מוצג, נעילת תוכן בזמן.

## 6. שמע — ההחלטה הארכיטקטונית החשובה ביותר

איכות הצליל של הברה בודדת ("בָּ") קריטית וה-Web Speech API לא מספיק טוב לזה. לכן:

1. **Build-time TTS:** סקריפט Node (`scripts/generate-audio.mjs`) שמריץ `edge-tts` (חינמי) עם קול `he-IL-HilaNeural`, מייצר MP3 לכל פריט תוכן (הברות, מילים, משפטים, הוראות קוליות) ושומר ב-`public/audio/{hash}.mp3`. הקבצים נכנסים לריפו — בזמן ריצה אין תלות ברשת חיצונית.
   - להברות בודדות: לייצר את הצליל ממילת עזר או מהוספת ה' עיצורית ("בָּה") אם ההגייה הישירה יוצאת גרועה — הסקריפט צריך לתמוך ב-`spokenForm` נפרד מ-`displayForm` בכל פריט תוכן.
2. **Fallback בזמן ריצה:** אם קובץ שמע חסר — `speechSynthesis` עם קול he-IL.
3. **מצב הקלטה הורית (v1.1, לא v1):** ההורה יכול להקליט את קולו במקום ה-TTS.
4. כל הוראות הממשק לילד מוקראות (אין הסתמכות על קריאת הוראות — הילד עוד לא קורא!). אייקון 🔊 בכל מסך משמיע שוב.

- `prefers-reduced-motion` מכובד.

## 8. מודל התוכן (JSON Schemas)

```ts
// src/content/types.ts
export type Nikud = 'kamatz'|'patach'|'hirik'|'hirikMale'|'tzere'|'segol'
                  |'holam'|'holamMale'|'shuruk'|'kubutz'|'shvaNa'|'shvaNach';

export interface SyllableItem {
  id: string;               // "bet-kamatz"
  display: string;          // "בָּ"  (תווי יוניקוד עם ניקוד)
  spokenForm?: string;      // "בָּה" — מה שנשלח ל-TTS אם שונה מהתצוגה
  letter: string;           // "ב"
  nikud: Nikud;
  distractors: string[];    // ids של מסיחים פונטית-דומים
}

export interface WordItem {
  id: string;
  display: string;          // "אַבָּא"
  syllables: string[];      // ["אַ","בָּ","א"]
  emoji?: string;           // "👨" — לתרגיל מילה-לתמונה
  frequencyRank?: number;   // למילים שכיחות
}

export interface SentenceItem {
  id: string;
  display: string;                       // משפט מנוקד מלא
  comprehension?: { question: string;    // מוקראת בשמע
                    options: string[];   // 3 אפשרויות (אימוג'י או מילים)
                    correctIndex: number };
}

export interface ParagraphItem {
  id: string;
  title: string;
  sentences: string[];                   // כל משפט = יחידת אישור הורה
  comprehension: SentenceItem['comprehension'][]; // 1–2 שאלות
}

export interface Level {
  id: number;               // 1–12
  name: string;             // "קמץ ופתח"
  icon: string;             // אימוג'י לתחנה במפה
  exerciseTypes: ('hearPick'|'seeHear'|'buildWord'|'wordToPic'|'readAloud')[];
  itemIds: string[];
  mixFromLevels: number[];  // ערבוב מצטבר משלבים קודמים (למשל שלב 3 מערבב 1–2)
  skipTestSize: 10;
  skipTestPassScore: 9;
}
```

**היקף תוכן v1 (חובה, נכתב ידנית — לא לגנרט מילים שלא בודקו):**
- שלבים 1–6: כל צירופי אות×ניקוד רלוונטיים (~180 הברות), כולל הבחנת בּ/ב, כּ/כ, פּ/פ, שׁ/שׂ.
- שלב 7: 60 מילים דו-הברתיות עם אימוג'י.
- שלב 8: 80 מילים ארוכות + 100 מילים שכיחות.
- שלבים 9–10: 40 משפטים (מנוקדים במלואם, כולל דגשים).
- שלבים 11–12: 12 פסקאות (נושאים: חיות, משפחה, כדורגל, ים — מותאם לבן 8).
- **כלל ברזל:** כל התוכן מנוקד ניקוד מלא ותקין. בכל ספק — לבדוק מול מילון. שגיאת ניקוד = באג חוסם.

## 9. מסכים

1. **מפת השלבים (בית):** שביל עם 12 תחנות, הדמות עומדת על השלב הנוכחי, כוכבים מוצגים על כל תחנה, כפתור ⚡ למבחן קפיצה על השלב הבא.
2. **מסך היכרות צליל:** ההברה ענקית במרכז, לחיצה משמיעה. "אָ… אָ… אָ…" — בדיוק כמו פעם, רק שהמסך מדבר.
3. **מסך סיבוב (תרגיל):** אזור שאלה למעלה (🔊 + תמונה אם יש), קלפי תשובה ענקיים, סולם התקדמות של הדמות בצד, ספירת פריטים (●●●○○○).
4. **מסך סיום סיבוב:** כוכבים, קונפטי, "עוד סיבוב?" / "חזרה למפה".
5. **מסך הורה (כניסה בלחיצה ארוכה 3 שניות על ⚙️):** גרף התקדמות לפי שלב, דיוק, זמן תרגול שבועי, רצף ימים, כפתור איפוס, ייצוא/ייבוא התקדמות (קובץ JSON).

### כיוון עיצובי (brief ל-frontend)
שפה חמה ומעודדת של "שעת סיפור", לא "אפליקציית דוחות": רקע שמיים בהיר בשעת בוקר, קלפים בצבעי גיר (chalk pastels) עם צל רך, הדמות והסולם הם החתימה הוויזואלית של המוצר — הם מופיעים בכל מסך תרגול ומטפסים פיזית עם כל הצלחה. טיפוגרפיה: התוכן הנלמד הוא הגיבור — ענק, ממורכז, עם ניקוד ברור; כל השאר קטן ושקט. אנימציה אחת מתוזמרת (הטיפוס+קונפטי), לא אפקטים מפוזרים.

## 10. לוגיקת הליבה — תור אדפטיבי (חובה לכסות ב-unit tests)

```
בתחילת סיבוב: queue = בחר 8–12 פריטים (חדשים + פריטים עם היסטוריית שגיאות מהעבר, ביחס 70/30)
לולאת סיבוב:
  הצג פריט → תשובה
  נכון בניסיון ראשון: item.strength += 1; הדמות מטפסת
  שגוי: השמע שוב, הדגש נכון, item.strength = 0,
        הכנס את הפריט חזרה לתור במרחק 2–3 פריטים (לא מיד)
פריט נחשב "נשלט" כש-strength ≥ 2 בשני סשנים שונים
שלב מושלם כש-≥90% מהפריטים נשלטו, או במבחן קפיצה 9/10
רצף 8 נכונים ראשונים בשלב ← מסומן fastTrack, מדלגים על 50% מהפריטים שטרם הוצגו
```

## 11. תוכנית מימוש ל-Claude Code — עבודה בלולאות

> **הנחיה כללית ל-Claude Code:** עבוד פאזה-פאזה. בכל פאזה: (1) ממש, (2) הרץ `npm run test && npm run build`, (3) אם נכשל — תקן וחזור ל-(2), **בלולאה עד שהכול ירוק**, (4) עדכן `PROGRESS.md` עם ✅ לכל קריטריון קבלה שהושג, (5) עבור לפאזה הבאה רק כשכל הקריטריונים של הנוכחית מסומנים. אל תבקש אישור בין פאזות. אם החלטה לא מכוסה במסמך — בחר את הפתרון הפשוט ביותר ותעד ב-`DECISIONS.md`.

### פאזה 0 — שלד (יום)
- Vite+React+TS+Tailwind+Zustand+Router, `dir="rtl"`, פונט עברי self-hosted, Vitest+Playwright מוגדרים, דיפלוי Vercel עובד.
- **קבלה:** `npm run dev/build/test` ירוקים; `package.json` בשם `hebread`; דף בית מציג את הלוגו "קוֹרֵא עִבְרִית" בפונט מנוקד תקין.

### פאזה 1 — מודל תוכן + שלבים 1–2
- types.ts, קובצי JSON לשלבים 1–2 (כל האותיות × קמץ/פתח/חיריק), ולידציית schema בסקריפט `npm run validate:content`.
- **קבלה:** הוולידציה עוברת; unit test שמוודא שלכל פריט יש ≥2 מסיחים קיימים.

### פאזה 2 — צינור השמע
- `scripts/generate-audio.mjs` עם edge-tts, מיפוי id→קובץ, hook `useAudio` עם preload ו-fallback ל-speechSynthesis.
- **קבלה:** לכל פריט תוכן קיים MP3; טסט שסורק ומאתר פריטים ללא שמע ונכשל אם יש.

### פאזה 3 — לולאת המשחק
- תור אדפטיבי (סעיף 10) כמודול טהור + unit tests מלאים; מסך תרגיל שמע-ובחר; מסך היכרות צליל; דמות+סולם; מסך סיום סיבוב; שמירת התקדמות.
- **קבלה:** Playwright: משתמש חדש משלים סיבוב מלא בשלב 1 כולל טעות אחת שחוזרת; רענון דף משמר התקדמות.

### פאזה 4 — מפה, קידום ומבחן קפיצה
- מפת 12 השלבים, לוגיקת השלמה/fastTrack/skip test, ערבוב מצטבר.
- **קבלה:** unit tests לכל כללי הקידום; Playwright: מעבר משלב 1 ל-2 דרך מבחן קפיצה.

### פאזה 5 — תוכן מלא שלבים 3–8 + תרגילי מילים
- כל ה-JSON לשלבים 3–8, תרגילי הרכב-מילה ומילה-לתמונה, סבב טורבו.
- **קבלה:** ולידציה+שמע לכל התוכן; Playwright לתרגיל הרכבת מילה.

### פאזה 6 — משפטים, פסקאות ומצב הורה
- שלבים 9–12, מסך קריאה-בקול עם ✓/✗ להורה, שאלות הבנה מוקראות, מסך הורה מלא כולל ייצוא/ייבוא.
- **קבלה:** Playwright: זרימת פסקה מלאה כולל אישורי הורה ושאלת הבנה.

### פאזה 7 — ליטוש ושחרור
- PWA/אופליין, `prefers-reduced-motion`, בדיקת ניקוד ויזואלית (צילומי מסך של כל ההברות), Lighthouse ≥90 בנגישות, README (כולל: איך מוסיפים תוכן, איך מריצים generate-audio), רישיון MIT, CONTRIBUTING.md.
- **קבלה:** build נקי, כל הטסטים ירוקים, Lighthouse עובר, האתר חי ב-Vercel.

## 12. מדדי הצלחה

- **המדד האמיתי:** תוך 8 שבועות של ~15 דק'/יום — הילד קורא פסקה של 40+ מילים מנוקדות ברצף (שלב 12 מושלם).

- טכני: 100% מהתוכן עם שמע תקין; אפס שגיאות ניקוד מדווחות.
