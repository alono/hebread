# hebread · קוֹרֵא עִבְרִית 🍞

A free, open-source web app that teaches Hebrew reading to kids (ages ~7–9) using
the **systematic phonics** method with **full niqqud**. It runs entirely in the
browser — no server, no accounts, no data collection — and works offline as a PWA.

A child opens it alone, hears a sound, picks the matching spelling, earns stars,
and climbs a 12-station ladder — from a single vowelled letter (**אָ**) all the
way to reading a full paragraph aloud.

- **On-screen name (for the child):** קוֹרֵא עִבְרִית — always in niqqud-ed Hebrew.
- **Method:** explicit letter+niqud→sound, phonemic awareness first (hear → choose),
  short sessions, spaced repetition, and real read-aloud text at the top levels.
- **Stack:** Vite + React 18 + TypeScript + Tailwind + Zustand + React Router. RTL.
- **Audio:** prebuilt MP3s (edge-tts, voice `he-IL-HilaNeural`), committed to the repo;
  a `speechSynthesis` fallback covers any gap at runtime.

The single source of truth for scope and pedagogy is [`.claude/PRD-hebread.md`](.claude/PRD-hebread.md).

## Quick start

```bash
npm install
npm run dev            # http://localhost:5173
```

## Scripts

| command | what it does |
|---|---|
| `npm run dev` | Vite dev server |
| `npm run build` | type-check (`tsc --noEmit`) + production build to `dist/` |
| `npm run preview` | serve the production build |
| `npm run test` | Vitest unit tests (game logic, content invariants, audio coverage) |
| `npm run test:e2e` | Playwright end-to-end flows |
| `npm run validate:content` | validate every content JSON against the schema + niqqud rules |
| `npm run generate:audio` | (re)generate MP3s for all content via edge-tts |

**Before committing any change, all three of these must pass:**

```bash
npm run test && npm run validate:content && npm run build
```

## Project layout

```
src/
  content/            # all learning content (JSON) + the shared schema (types.ts)
    syllables/        # levels 1–6  (generated by scripts/gen-syllables.mjs)
    words/            # levels 6–8  (hand-authored)
    sentences/        # levels 9–10 (hand-authored)
    paragraphs/       # levels 11–12 (hand-authored)
    audio-manifest.json  # id → { file, text } (generated)
    index.ts          # loads content, builds the 12 Level objects
    levels.ts         # per-level metadata (names, icons, exercise mix)
  game/               # PURE, unit-tested logic: adaptive queue, round, progress, promotion
  components/         # UI pieces (companion, ladder, cards, exercises…)
  screens/            # Map, PlayLevel, SkipTest, Turbo, ReadAloud, ParentMode
  hooks/useAudio.ts   # MP3 playback + preload + speechSynthesis fallback
public/audio/         # committed MP3s (served statically, precached for offline)
scripts/              # generators + validator
```

## Adding or editing content

All child-facing text must be **Hebrew, fully and correctly niqqud-ed** — a niqqud
error is a blocking bug. The workflow (see also `.claude/skills/add-content`):

1. **Syllables (levels 1–6)** are generated. Edit the letter table / niqud / distractor
   map in `scripts/gen-syllables.mjs`, then:
   ```bash
   node scripts/gen-syllables.mjs
   ```
   **Words / sentences / paragraphs** (levels 6–12) are hand-authored — edit the JSON in
   `src/content/{words,sentences,paragraphs}/` following the `types.ts` schemas.
   - For words, `syllables` must concatenate exactly to `display` (glyph-boundary split);
     the validator enforces this.
   - Comprehension `options` may be emoji (shown, not spoken) or niqqud-ed Hebrew (spoken).
2. Validate: `npm run validate:content` — fix until green. It checks the schema,
   NFC-canonical niqqud order, and that every distractor / syllable / index is valid.
3. Generate audio (see below): `npm run generate:audio`.
4. Have the niqqud reviewed (the project uses a dedicated `nikud-reviewer` review step).
5. `npm run test && npm run build`.

### Generating audio (edge-tts)

Audio is built once and committed, so you only need this when you add/change content.
It uses Microsoft's free `edge-tts` (Python), which needs network access at build time.

```bash
python3 -m venv .venv
.venv/bin/pip install edge-tts
npm run generate:audio          # writes public/audio/*.mp3 + audio-manifest.json
node scripts/generate-audio.mjs --check   # CI-friendly: fails if any item lacks audio
```

The script is content-addressed (files are named by a hash of the spoken text), so it
only synthesises what changed and prunes clips no longer referenced.

### Visual niqqud check

To eyeball every syllable rendered in the shipping font:

```bash
node scripts/niqqud-screenshot.mjs   # writes niqqud-check.png (requires a built dist/ + Playwright)
```

## Deploying (Vercel)

The app is fully static. `vercel.json` is already configured (framework `vite`,
output `dist/`, SPA rewrite, zero env vars). Deploy with:

```bash
npm i -g vercel   # once
vercel            # preview
vercel --prod     # production
```

or connect the repo in the Vercel dashboard and accept the detected settings.
